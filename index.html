§ 01

System Architecture

Natflay follows a modular monolith-to-microservices evolution path. The MVP launches as a well-bounded monolith on Next.js + Node, then splits hot paths (AI, webhooks, notifications) into dedicated services as traffic scales.

Core Modules

Four tightly-scoped modules, each a bounded context with its own API surface, data model, and integration surface.

Module A MVP

Omnichannel Sales & Store Management

Connect, sync, and operate multiple storefronts from a single pane of glass.

Priority
P0

✓ Integrations

  • ▸ Shopify Admin API (REST + GraphQL) via OAuth
  • ▸ WooCommerce REST API (Phase 2)
  • ▸ Custom webhook receiver for generic stores
  • ▸ Per-store credential vault (AES-256 encrypted)

⚙ Core Functionality

  • ▸ Real-time product, inventory & order sync
  • ▸ Remote order management (view / update / refund)
  • ▸ Unified order timeline across stores
  • ▸ Webhook-driven delta sync (no polling)
★ Pro Feature
Bulk Operations & Cross-Channel Inventory Alerts

Bulk-update order statuses (fulfilled, cancelled, refunded) across multiple stores in one action. Configurable low-stock alerts propagate across all connected channels, preventing oversell.

Context-aware AI agents that know your business, your products, and your rules — across every messaging channel.

P0

Incoming Message → Channel Adapter → Normalize → Retrieve RAG Context → LLM w/ Tools → Guardrails → Reply + Log
★ Pro Feature
Human-in-the-Loop Handoff

When the AI's confidence drops below threshold, or the user explicitly asks for a human, the conversation is flagged and routed to a live agent inbox inside Natflay. Agents see full context: transcript, retrieved KB chunks, customer order history, and suggested replies.

§ 07

Development Roadmap

Three MVP phases, each delivering a shippable slice of value. Target: MVP live in 14 weeks with a 3-engineer core team.

Phase 1 Weeks 1–4 Foundation & Shopify Sync

Get the bones in place and prove the core value: connecting a Shopify store and seeing orders flow.

  • ✓ Auth (Clerk) + multi-tenant workspace model
  • ✓ Shopify OAuth integration
  • ✓ Product & order sync (webhook-driven)
  • ✓ Dashboard shell (Next.js + shadcn)
  • ✓ Orders list with filters (status, country, date)
  • ✓ CSV export (shipping-agent format)
  • ✓ Basic RBAC (owner + member)
  • ✓ CI/CD + staging environment
Deliverable: A merchant can connect Shopify, view orders, and export CSVs for their shipping agent.
Phase 2 Weeks 5–9 AI Support & Fulfillment Automation

Turn Natflay into a true automation hub — AI answers customers, tracking flows back to Shopify.

  • ✓ Knowledge Base CRUD + embedding pipeline
  • ✓ RAG chat engine (GPT-4o + Pinecone)
  • ✓ Meta Messenger + Instagram DM integration
  • ✓ WhatsApp Cloud API integration
  • ✓ Telegram Bot integration
  • ✓ Unified inbox with human handoff
  • ✓ Tracking push-back to Shopify
  • ✓ Automated WhatsApp/Email tracking notifications
Deliverable: AI answers 70%+ of customer queries across 4 channels. Tracking updates flow end-to-end.
Phase 3 Weeks 10–14 Store Builder & Pro Features

Launch the native storefront engine and unlock the premium tier that makes Natflay a full operating system.

  • ✓ Block-based landing page editor
  • ✓ COD-optimized checkout
  • ✓ Custom domain mapping + auto SSL
  • ✓ Store analytics (visits, conversion, revenue)
  • ✓ Unified financial dashboard (Revenue – Ads – Shipping = Profit)
  • ✓ Meta Marketing API ad-spend sync
  • ✓ Telegram + in-app alert system
  • ✓ Stripe subscriptions (Free / Pro / Enterprise)
Deliverable: A merchant can launch a storefront, run their business, and see real-time profit — all inside Natflay.
§ 08

Security, Scale & DevOps

Non-negotiables for a platform that holds customer data, payment credentials, and business secrets.

️ Security

  • ▸ TLS 1.3 everywhere · HSTS preload
  • ▸ AES-256-GCM encryption for OAuth tokens at rest
  • ▸ Row-Level Security on every tenant-scoped table
  • ▸ Webhook signature verification (HMAC-SHA256)
  • ▸ Rate limiting per user + per IP (Redis)
  • ▸ Audit log for sensitive actions (immutable)
  • ▸ Secrets via AWS Secrets Manager / Doppler
  • CSP, CORS, CSRF hardened

📈 Scale Targets

  • ▸ 10k workspaces at launch · horizontal scale to 100k
  • ▸ 500 orders/sec ingestion peak (BullMQ back-pressure)
  • ▸ 1M messages/day across AI channels
  • ▸ p95 API latency < 250ms (non-AI)
  • ▸ p95 AI reply latency < 3s
  • ▸ 99.9% uptime SLO (MVP) → 99.95% (Pro)

🚀 DevOps

  • ▸ Monorepo (Turborepo) · apps + packages
  • ▸ Docker images · multi-stage builds
  • ▸ GitHub Actions · preview deploys per PR
  • ▸ Staging mirrors production (isolated DB)
  • ▸ Blue/green deploys on AWS ECS
  • ▸ DB migrations via Prisma Migrate · zero-downtime

📊 Observability

  • ▸ Sentry · error tracking + performance
  • ▸ PostHog · product analytics
  • ▸ OpenTelemetry · distributed tracing
  • ▸ Grafana + Prometheus · infra metrics
  • ▸ LLM cost dashboard (tokens · $/workspace)
  • ▸ PagerDuty · on-call rotation
